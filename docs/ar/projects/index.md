# مرحبًا بك في المشاريع

هذا هو القسم العملي من Learn Harness Engineering. قراءة المحاضرات وحدها لا تكفي؛ عليك بناء البيئات بنفسك ومراقبة كيف يتصرف Codex أو Claude Code أو غيرهما من وكلاء الذكاء الاصطناعي تحت قواعد مختلفة.

## نظرة عامة على المشاريع

يضم هذا المساق 6 مشاريع عملية متدرجة تعلمك كيف تبني بيئة عمل وكيلية موثوقة من الصفر:

1. **Prompt فقط مقابل القواعد أولًا**: قارن أداء الوكيل عند استخدام prompt فقط مقابل harness أساسي.
2. **Workspace قابل للقراءة من الوكيل**: تعلم كيف تنظم المستودع ليكون مناسبًا للذكاء الاصطناعي، وكيف تنشئ آليات handoff.
3. **استمرارية متعددة الجلسات**: صمم ملفات حالة وسكربتات تهيئة حتى يستطيع الوكيل استئناف العمل بسلاسة بين الجلسات.
4. **Feedback في runtime والتحكم في النطاق**: أدخل أدوات تسمح للوكيل باختبار كوده وتصحيح الأخطاء أثناء التنفيذ.
5. **التحقق الذاتي وفصل الأدوار**: ابن آلية مراجعة مستقلة لمنع الهلوسة وإعلان النجاح المبكر.
6. **Harness كامل (Capstone)**: اجمع بيئة عمل وكيلية نهائية قابلة للملاحظة وتعمل end-to-end.

## كيفية المتابعة

عادة يحتوي كل مجلد مشروع على:
- `starter/`: مساحة العمل التي تبدأ منها.
- `solution/`: تنفيذ مرجعي إذا علقت.
- تعليمات مهمة توضح الخلفية والأهداف المحددة.

استخدم AI Coding Agent المفضل لديك، مثل Claude Code أو Cursor أو Trae، لإكمال المهام داخل مجلد `starter/`.
