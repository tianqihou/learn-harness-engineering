# Anti-Patterns für Instruktionsdateien

- Alles Repository-Wissen in eine Datei packen
- Dieselbe Regel an mehreren Stellen wiederholen
- Veraltete Regeln kodieren, die niemand prüft
- So spezifische bedingte Anweisungen schreiben, dass sie selten zutreffen
- Lange Tool-Handbücher in den Startup-Kontext einbetten
