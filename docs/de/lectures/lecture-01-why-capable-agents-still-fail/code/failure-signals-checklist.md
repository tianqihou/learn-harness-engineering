# Fehlersignal-Checkliste

Verwenden Sie diese Checkliste bei der Überprüfung eines schwachen Harness-Laufs.

- Hat der Agent gefragt oder falsch abgeleitet, wie die App gestartet wird?
- Hat er Verzeichnisse oder Abstraktionen erstellt, die nicht zum intended Produkt passen?
- Hat er nach Erstellung einer sichtbaren UI-Hülle ohne vollständigen Workflow aufgehört?
- Hat er Notizen oder Artefakte hinterlassen, die einem zukünftigen Lauf helfen?
- Könnte eine frische Session in unter fünf Minuten verstehen, was passiert ist?
